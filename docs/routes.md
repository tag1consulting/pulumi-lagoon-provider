# Managing Lagoon Routes with Pulumi

This guide covers how to manage Lagoon API-managed routes using the Pulumi Lagoon provider.

## Overview

Starting with Lagoon v2.29.0, routes can be managed directly through the Lagoon API. This allows you to define routes as infrastructure-as-code alongside your projects and environments.

**Requirements:**
- Lagoon v2.29.0 or later
- `featureApiRoutes` enabled on the Lagoon Organization

API-managed routes support:
- **TLS configuration** - ACME certificates, HSTS settings
- **Path routing** - Route specific paths to different services (up to 10 path rules)
- **Alternative names** - Additional domain aliases (up to 25)
- **Annotations** - Custom ingress annotations
- **Monitoring** - Health check endpoints for uptime monitoring
- **Request verification** - Control ingress request verification

This provider supports three route-related resources:
- **Route** - Individual API-managed routes attached to projects and optionally to environments
- **ProjectAutogeneratedRouteConfig** - Project-level autogenerated route configuration
- **EnvironmentAutogeneratedRouteConfig** - Environment-level autogenerated route configuration

## Route Resource

Manages an API-managed route attached to a project and optionally to a specific environment.

### Basic Example

```python
from pulumi_lagoon import Route, RouteArgs

route = Route("main-domain",
    RouteArgs(
        project_name="my-drupal-site",
        domain="www.example.com",
        service="nginx",
        environment="main",
        tls_acme=True,
        insecure="Redirect",
    )
)
```

### Complete Example with All Features

```python
from pulumi_lagoon import Route, RouteArgs

route = Route("main-route",
    RouteArgs(
        project_name="my-drupal-site",
        domain="example.com",
        service="nginx",
        environment="main",
        
        # TLS configuration
        tls_acme=True,
        insecure="Redirect",
        
        # HSTS security headers
        hsts_enabled=True,
        hsts_max_age=31536000,
        hsts_preload=True,
        hsts_include_subdomains=True,
        
        # Route type and priority
        route_type="ACTIVE",
        primary=True,
        
        # Monitoring
        monitoring_path="/health",
        
        # Ingress settings
        disable_request_verification=False,
        
        # Alternative domains
        alternative_names=["www.example.com", "example.org"],
        
        # Path-based routing to different services
        path_routes=[
            {"path": "/api", "to_service": "api"},
            {"path": "/admin", "to_service": "admin-panel"},
        ],
        
        # Custom annotations
        annotations=[
            {"key": "nginx.ingress.kubernetes.io/proxy-body-size", "value": "50m"},
            {"key": "cert-manager.io/cluster-issuer", "value": "letsencrypt-prod"},
        ],
    )
)
```

### Properties

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `project_name` | `str` | Yes | Project name (force-new) |
| `domain` | `str` | Yes | Domain name (force-new, unique identifier) |
| `service` | `str` | No | Service to route traffic to |
| `environment` | `str` | No | Environment to attach route to (if omitted, project-level) |
| `tls_acme` | `bool` | No | Enable ACME TLS certificate |
| `insecure` | `str` | No | Insecure behavior: `Allow`, `Redirect`, or `None` |
| `route_type` | `str` | No | Route type: `STANDARD`, `ACTIVE`, or `STANDBY` |
| `primary` | `bool` | No | Whether this is the primary route |
| `monitoring_path` | `str` | No | Path for uptime monitoring (e.g., `/health`) |
| `disable_request_verification` | `bool` | No | Disable ingress request verification |
| `hsts_enabled` | `bool` | No | Enable HSTS headers |
| `hsts_max_age` | `int` | No | HSTS max-age in seconds (default: 31536000) |
| `hsts_preload` | `bool` | No | Include in HSTS preload list |
| `hsts_include_subdomains` | `bool` | No | Apply HSTS to subdomains |
| `annotations` | `list` | No | List of `{key, value}` annotation objects (unlimited) |
| `alternative_names` | `list` | No | List of alternative domain names (max 25) |
| `path_routes` | `list` | No | List of `{path, to_service}` path routing rules (max 10) |

### Outputs

- `project_name` - The project name
- `domain` - The domain name
- `service` - The service name
- `environment` - The environment name (if set)
- `tls_acme` - TLS ACME enabled status
- All other input properties are also available as outputs

### Resource ID

`{project_name}:{domain}`

**Note:** Both `project_name` and `domain` are force-new fields. Changing either will recreate the route.

### Important: monitoringPath Limitation

Due to a Lagoon API limitation, `monitoringPath` cannot be set during route creation. The provider uses a create-then-update pattern:

1. Create the route without `monitoringPath`
2. Immediately update it with the `monitoringPath` value

If the follow-up update fails, the provider:
- Emits a Pulumi warning
- Returns success with `monitoringPath=nil` in state (to avoid orphaning the route)
- Allows the next `pulumi up` to retry via the normal Update path

## ProjectAutogeneratedRouteConfig Resource

Manages the autogenerated route configuration at the project level. This resource uses upsert semantics — create and update use the same mutation.

### Example

```python
from pulumi_lagoon import ProjectAutogeneratedRouteConfig, ProjectAutogeneratedRouteConfigArgs

config = ProjectAutogeneratedRouteConfig("project-routes",
    ProjectAutogeneratedRouteConfigArgs(
        project_name="my-drupal-site",
        enabled=True,
        allow_pull_requests=True,
        prefixes=["www", "dev"],
        tls_acme=True,
        insecure="Redirect",
        disable_request_verification=False,
        path_routes=[
            {"path": "/api", "to_service": "api"},
        ],
    )
)
```

### Properties

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `project_name` | `str` | Yes | Project name (force-new) |
| `enabled` | `bool` | No | Enable autogenerated routes |
| `allow_pull_requests` | `bool` | No | Allow autogenerated routes for pull requests |
| `prefixes` | `list[str]` | No | List of domain prefixes |
| `path_routes` | `list` | No | List of `{path, to_service}` path routing rules |
| `disable_request_verification` | `bool` | No | Disable request verification |
| `insecure` | `str` | No | Insecure behavior: `Allow`, `Redirect`, or `None` |
| `tls_acme` | `bool` | No | Enable ACME TLS |

### Resource ID

`{project_name}`

## EnvironmentAutogeneratedRouteConfig Resource

Manages the autogenerated route configuration at the environment level.

### Example

```python
from pulumi_lagoon import EnvironmentAutogeneratedRouteConfig, EnvironmentAutogeneratedRouteConfigArgs

env_config = EnvironmentAutogeneratedRouteConfig("main-routes",
    EnvironmentAutogeneratedRouteConfigArgs(
        project_name="my-drupal-site",
        environment_name="main",
        enabled=True,
    )
)
```

### Properties

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `project_name` | `str` | Yes | Project name (force-new) |
| `environment_name` | `str` | Yes | Environment name (force-new) |
| `enabled` | `bool` | No | Enable autogenerated routes for the environment |

### Resource ID

`{project_name}:{environment_name}`

## Complete Multi-Route Example

Here's a comprehensive example showing multiple routes for a production site:

```python
import pulumi
from pulumi_lagoon import (
    Project, ProjectArgs,
    Environment, EnvironmentArgs,
    Route, RouteArgs,
    ProjectAutogeneratedRouteConfig, ProjectAutogeneratedRouteConfigArgs,
    EnvironmentAutogeneratedRouteConfig, EnvironmentAutogeneratedRouteConfigArgs,
)

# Create project
project = Project("drupal-site",
    ProjectArgs(
        name="drupal-site",
        git_url="git@github.com:example/drupal-site.git",
        deploytarget_id=1,
        production_environment="main",
        branches="^(main|develop|stage)$",
    )
)

# Create production environment
prod_env = Environment("production",
    EnvironmentArgs(
        name="main",
        project_id=project.lagoon_id,
        deploy_type="branch",
        deploy_base_ref="main",
        environment_type="production",
    )
)

# Primary production route with all features
primary_route = Route("primary",
    RouteArgs(
        project_name=project.name,
        domain="example.com",
        service="nginx",
        environment="main",
        tls_acme=True,
        insecure="Redirect",
        route_type="ACTIVE",
        primary=True,
        monitoring_path="/health",
        hsts_enabled=True,
        hsts_max_age=31536000,
        hsts_preload=True,
        hsts_include_subdomains=True,
        alternative_names=["www.example.com"],
        path_routes=[
            {"path": "/api", "to_service": "api"},
            {"path": "/admin", "to_service": "admin-panel"},
        ],
        annotations=[
            {"key": "nginx.ingress.kubernetes.io/proxy-body-size", "value": "100m"},
        ],
    ),
    opts=pulumi.ResourceOptions(depends_on=[project, prod_env])
)

# Standby route for failover
standby_route = Route("standby",
    RouteArgs(
        project_name=project.name,
        domain="standby.example.com",
        service="nginx",
        environment="main",
        tls_acme=True,
        route_type="STANDBY",
    ),
    opts=pulumi.ResourceOptions(depends_on=[project, prod_env])
)

# Configure autogenerated routes for non-production branches
autogen_config = ProjectAutogeneratedRouteConfig("autogen",
    ProjectAutogeneratedRouteConfigArgs(
        project_name=project.name,
        enabled=True,
        allow_pull_requests=True,
        prefixes=["dev", "staging"],
        tls_acme=True,
        insecure="Redirect",
    ),
    opts=pulumi.ResourceOptions(depends_on=[project])
)

# Override autogen behavior for main environment
env_autogen = EnvironmentAutogeneratedRouteConfig("main-autogen",
    EnvironmentAutogeneratedRouteConfigArgs(
        project_name=project.name,
        environment_name="main",
        enabled=False,  # Disable for production (using explicit routes)
    ),
    opts=pulumi.ResourceOptions(depends_on=[project, prod_env])
)

# Exports
pulumi.export("primary_domain", primary_route.domain)
pulumi.export("standby_domain", standby_route.domain)
```

## Importing Existing Routes

### Import ID Formats

| Resource | Format | Example |
|----------|--------|---------|
| `Route` | `{project_name}:{domain}` | `my-project:example.com` |
| `ProjectAutogeneratedRouteConfig` | `{project_name}` | `my-project` |
| `EnvironmentAutogeneratedRouteConfig` | `{project_name}:{env_name}` | `my-project:main` |

### Import Examples

```bash
# Import an API-managed route
pulumi import lagoon:index:Route main-route my-project:example.com

# Import project autogenerated route config
pulumi import lagoon:index:ProjectAutogeneratedRouteConfig project-config my-project

# Import environment autogenerated route config
pulumi import lagoon:index:EnvironmentAutogeneratedRouteConfig env-config my-project:main
```

After importing, add the corresponding resource definition to your Pulumi code:

```python
from pulumi_lagoon import Route, RouteArgs

# After importing "my-project:example.com"
route = Route("main-route",
    RouteArgs(
        project_name="my-project",
        domain="example.com",
        service="nginx",
        environment="main",
        tls_acme=True,
        insecure="Redirect",
    ),
    opts=pulumi.ResourceOptions(import_="my-project:example.com")
)
```

## Validation Rules

### Domain Names
- Must be valid DNS names
- Cannot contain uppercase letters (Lagoon normalizes to lowercase)
- Must be unique within the project

### Path Routes
- Maximum 10 path routing rules per route
- Paths must start with `/`
- `to_service` must be a valid service name

### Alternative Names
- Maximum 25 alternative domain names per route
- Each must be a valid DNS name

### Route Types
Valid values for `route_type`:
- `STANDARD` - Default route behavior
- `ACTIVE` - Active route in an active/standby pair
- `STANDBY` - Standby route for failover scenarios

### Insecure Behavior
Valid values for `insecure`:
- `Allow` - Allow both HTTP and HTTPS
- `Redirect` - Redirect HTTP to HTTPS
- `None` - HTTPS only

## Troubleshooting

### Route Creation Fails

1. **"Route already exists"** - Check if the route is managed elsewhere (API, UI, or .lagoon.yml). Only one management method can be used at a time.
2. **Feature not enabled** - Verify `featureApiRoutes` is enabled on your Lagoon Organization.
3. **Lagoon version too old** - API-managed routes require v2.29.0 or later.

### monitoringPath Not Set

If the create-then-update pattern fails for `monitoringPath`:
1. Check the Pulumi output for warnings
2. Verify the route was created (even without `monitoringPath`)
3. Run `pulumi up` again — the provider will retry setting `monitoringPath` via Update

### Domain Resolution Issues

1. **DNS not pointing to cluster** - Verify DNS A/CNAME records point to the Lagoon cluster's ingress controller
2. **Certificate not issued** - Check cert-manager logs if `tls_acme=true`
3. **Route not in ingress** - Check Kubernetes ingress resources in the environment namespace

### Import Fails

1. **"not found"** - Verify the route exists: `lagoon list routes --project my-project`
2. **Wrong format** - Use `project_name:domain` format (case-sensitive)
3. **Legacy .lagoon.yml route** - Only API-managed routes can be imported (not .lagoon.yml-based routes)
