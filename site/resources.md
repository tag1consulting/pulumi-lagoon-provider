# Resources

The Pulumi Lagoon Provider includes 18 resources organized into six categories. Each resource corresponds to an object managed through the Lagoon GraphQL API.

| Resource | Description | Documentation |
|----------|-------------|---------------|
| Project | Lagoon projects (applications/sites) | [Core Resources](resources/core) |
| Environment | Environments (branch/PR deployments) | [Core Resources](resources/core) |
| Variable | Project and environment variables | [Core Resources](resources/core) |
| Group | Groups for organizing projects and users | [Core Resources](resources/core) |
| DeployTarget | Kubernetes cluster deploy targets | [Deploy Targets](resources/deploy-targets) |
| DeployTargetConfig | Branch-pattern routing to deploy targets | [Deploy Targets](resources/deploy-targets) |
| NotificationSlack | Slack deployment notifications | [Notifications](resources/notifications) |
| NotificationRocketChat | RocketChat deployment notifications | [Notifications](resources/notifications) |
| NotificationEmail | Email deployment notifications | [Notifications](resources/notifications) |
| NotificationMicrosoftTeams | Microsoft Teams deployment notifications | [Notifications](resources/notifications) |
| ProjectNotification | Link notifications to projects | [Notifications](resources/notifications) |
| Route | API-managed routes (requires Lagoon v2.29.0+) | [Routes](resources/routes) |
| ProjectAutogeneratedRouteConfig | Project-level autogenerated route configuration | [Routes](resources/routes) |
| EnvironmentAutogeneratedRouteConfig | Environment-level autogenerated route configuration | [Routes](resources/routes) |
| Task | Advanced task definitions (command and image types) | [Tasks](resources/tasks) |
| User | Lagoon users | [Users](resources/users) |
| UserGroupAssignment | User-to-group role assignments | [Users](resources/users) |
| UserPlatformRole | Platform-level role assignments | [Users](resources/users) |

{: .note }
> The `lagoonId` output type varies across resources: most resources use an integer ID, but `Group` returns a UUID string and `User` returns a string ID. If you build automation that references resources by their Lagoon IDs, do not assume a single type.
